Product Overview

1.1 Vision

ChessIQ is an AI-powered personalized chess intelligence and coaching platform that transforms how players improve at chess. Unlike traditional game analyzers that provide one-off engine evaluations, ChessIQ builds a comprehensive understanding of each player's unique patterns, tendencies, and developmental needs across their entire game history.

ChessIQ Is NOTChessIQ IS
Just a game analyzerA personalized AI chess coach that learns your unique playing style
Just a Stockfish wrapperA behavioral pattern recognition system identifying recurring tendencies across hundreds of games
Just a dashboard for metricsA conversational chess intelligence platform where players ask questions and receive contextual, personalized guidance

The key differentiator is long-term pattern recognition: The AI learns recurring patterns across a player's games and provides personalized coaching based on those longitudinal insights, not just individual game analysis.

Target users: Intermediate online players (~1000–2200 Elo) seeking personalized, actionable coaching that evolves with their play.

1.2 High-Level Capabilities

Core Platform

  • Chess.com account linking by username (no OAuth in MVP)
  • Game fetching with filters: count, time control, rated/unrated, date range
  • Chess.com is the external source of truth for raw game data (PGNs, move history, ratings, timestamps)
  • Lightweight game storage — recently fetched games are temporarily cached; Stockfish analysis outputs and detected patterns are stored permanently
  • Batch analysis of games using Stockfish via Celery workers
  • Move timing and clock data ingestion for behavioral analysis
  • Analysis depth advantage: ChessIQ's free-tier analysis depth may exceed Chess.com's free analysis, framed as deeper analysis + personalized coaching + longitudinal intelligence

Pattern Recognition Engine

  • Long-term behavioral pattern detection across hundreds of games
  • Recurring mistake identification in similar positions
  • Opening-specific weakness detection, middlegame collapse patterns, tactical recurrence
  • Endgame weakness profiling, strategic habit detection
  • Time management issue identification, hesitation/overthinking detection
  • Impulsive play pattern recognition and longitudinal player profile generation

AI Coach / Conversational Interface

  • Natural language Q&A about player patterns and games
  • Personalized improvement recommendations and context-aware coaching
  • Interactive drill and study suggestions

User Roles & Personas

2.1 Roles

  • Player (End User) — Owns a profile, links Chess.com username, fetches games, runs analyses, consumes recommendations and training drills
  • Admin / Support — Inspects user stats and logs, triggers re-analysis, adjusts system-wide settings
  • System / Background Workers — Celery workers performing heavy engine analysis and generating insights

2.2 Primary User Journeys

First-Time Setup

  1. Enters Chess.com username and email
  2. ChessIQ creates or finds user
  3. User chooses filters (e.g., last 25 rapid rated games)
  4. Games fetched, stored, and visible on dashboard as "Not analyzed"
  5. User clicks "Analyze All Games" to start analysis

Return Visit

  1. Lands directly on /dashboard?username=…
  2. Sees summary of performance and last analyzed period
  3. Can sync new games, re-analyze, or enter Training Mode

Functional Requirements

3.1 Authentication & Identity

FR-AUTH-1
Users authenticate via lightweight account (email + Chess.com username). No password in MVP.
FR-AUTH-2
System uniquely identifies users by lowercase chesscom_username.
FR-AUTH-3
On first login: If user exists → "Welcome back". Else → create Supabase users row with username, display name, email, timestamps.

3.2 Game Fetching

3.2.1 Filters & Options

FR-GAMES-1
User can choose: game_count (10, 25, 50, 100), time_controls (rapid, blitz, bullet, daily), rated_filter (rated, unrated, both), optional start_date/end_date.

3.2.2 Behavior

FR-GAMES-2
On new fetch, app uses replace all semantics: remove or logically archive prior games within scope. Chess.com is the external source of truth; ChessIQ maintains only the working cache needed for active analysis and coaching.
FR-GAMES-3
"Get Started" flow: Find/create user → POST games/fetch with filters → show progress → toast fetched count → redirect to /dashboard.
FR-GAMES-4
Backend ensures idempotent storage (no duplicate games per user_id + external game id).

3.3 Dashboard — Fetched Games

FR-DASH-GAMES-1
Dashboard displays "Fetched Games" section with title and {N} games subtext.
FR-DASH-GAMES-2
Each game card shows: opponent username, result (Win/Loss/Draw with icon), time control, rated/unrated, date/time, Chess.com link, badge (Not analyzed or Analyzed).
FR-DASH-GAMES-3
Section is collapsible (default expanded). Clicking header toggles state.
FR-DASH-GAMES-4
Top-right "Analyze All Games" button — disabled if analysis is running or all games already analyzed.

3.4 Dashboard — Analysis Summary & Insights

FR-ANALYSIS-1
Summary panel: total games analyzed, overall accuracy metric (0–100), Win/Draw/Loss distribution, ACPL trend or period comparison.
FR-ANALYSIS-2
Phase-based breakdown: Opening, Middlegame, Endgame scores (0–100 or A–E) with phase-level blunder/mistake/inaccuracy counts.
FR-ANALYSIS-3
Move-quality chart (pie or stacked bar): Best, Excellent, Good, Inaccuracy, Mistake, Blunder.

3.5 Coaching / Recommendations

FR-COACH-1
Coaching Insights panel lists 3–7 key insights with: Category, Headline, Description, Severity level (low/medium/high), Recommended drill type.
FR-COACH-2
Insights derived from: repeated blunders in similar positions, high ACPL or low accuracy in specific phases, particular tactical themes frequently missed.

3.6 Analysis Workflow

FR-WORKFLOW-1
"Analyze All Games" → frontend calls backend endpoint → backend queues Celery jobs for unanalyzed games → frontend shows modal with queued count.
FR-WORKFLOW-2
Frontend periodically polls summary/games status until all queued games show is_analyzed = true or timeout/max polling hits.
FR-WORKFLOW-3
On completion: modal closes, dashboard summary and game list update, toast notification fires.
FR-WORKFLOW-4
"Re-analyze" button for advanced users forces recompute of previously analyzed games.
FR-WORKFLOW-5
Premium Pro/Elite users can enable automatic background analysis: periodic Chess.com game fetching, auto-queue for Stockfish, incremental profile updates.
FR-WORKFLOW-6
Premium Auto-analysis generates scheduled reports (daily/weekly coaching summary) delivered as in-app notifications and optionally via email.

3.7 Training Mode

FR-TRAIN-1
Training Mode accessible from game rows ("Train this game") or insight cards ("Practice rook endgames").
FR-TRAIN-2
Training mode includes: interactive board, move list, engine evaluation bar (optional for MVP).
FR-TRAIN-3
Drill behavior: board shows position just before user's mistake → user chooses best move → correct: confirmation + explanation → incorrect: feedback + engine move + retry.
FR-TRAIN-4
Track number of drills attempted and success rate by theme.

3.8 Pattern Recognition Engine

The Pattern Recognition Engine is the intelligence layer that transforms individual game analyses into personalized coaching insights. It operates on the principle of longitudinal analysis — identifying recurring patterns across many games.

3.8.1 Pattern Detection Categories

FR-PATTERN-1
Recurring Positional Mistakes — Detect when user repeatedly blunders in similar position types. E.g.: "You have blundered material in 8 out of 15 King's Gambit middlegame positions with this pawn structure."
FR-PATTERN-2
Opening-Specific Weaknesses — Track performance by opening/repertoire line. E.g.: "You lose 73% of games when playing the Sicilian Dragon."
FR-PATTERN-3
Middlegame Collapse Patterns — Identify at-risk transition points. E.g.: "You consistently lose positional battles around moves 18-25."
FR-PATTERN-4
Tactical Recurrence — Frequently missed tactical themes (forks, skewers, pins). E.g.: "You've missed knight forks in similar positions 14 times in the last 50 games."
FR-PATTERN-5
Endgame Weaknesses — Recurring technique failures. E.g.: "You have drawn 6 winning rook endgames this month."
FR-PATTERN-6
Strategic Habits — Positional vs. tactical tendencies. E.g.: "You perform better in closed positions (62% win rate) vs. open positions (41% win rate)."
FR-PATTERN-7
Castling & King Safety — Delayed castling risks. E.g.: "Your win rate drops to 28% when you lose castling rights before move 15."
FR-PATTERN-8
Strong Recurring Patterns (Strengths) — Positions where user consistently excels. E.g.: "You have a 78% success rate in IQP positions."

3.8.2 Pattern Scoring & Severity

FR-PATTERN-9
Severity Levels: Critical >30% of relevant games, high impact · Significant 15–30% of relevant games · Developing <15% of games · Historical detected but improving
FR-PATTERN-10
Confidence Scoring — Based on: sample size (games analyzed), statistical significance, recency weighting.

3.8.3 Longitudinal Player Profile

FR-PATTERN-11
Player Archetype Generation — System generates a "player archetype" combining strengths, weaknesses, and stylistic tendencies. Examples: "The Tactician with Endgame Blind Spots", "The Solid Builder with Opening Gaps", "The Time Pressure Panicker".
FR-PATTERN-12
Evolution Tracking — Track how patterns change over time; surface "breakthrough" moments; identify persistent vs. resolved weaknesses.

3.9 Move Timing & Clock Data Analysis

ChessIQ analyzes not just what moves were played, but how time was used — revealing behavioral patterns invisible to traditional engine analysis.

3.9.1 Time Data Ingestion

FR-TIMING-1
Clock Data Capture — Ingest move timestamps from Chess.com PGNs; calculate think time per move; store clock remaining; track time spent by phase.
FR-TIMING-2
Time Pressure Detection — Identify moves under time pressure (<30s blitz, <2min rapid); flag "panic moves"; correlate with move quality.

3.9.2 Behavioral Pattern Detection

FR-TIMING-3
Overthinking Patterns — Detect excessive think time followed by blunders. E.g.: "You spent 3+ minutes on 8 positions and blundered in 6."
FR-TIMING-4
Impulsive Play Detection — Flag moves played too quickly given position complexity. E.g.: "You blundered on 12 moves played in under 5 seconds when you had plenty of time."
FR-TIMING-5
Time Management Collapse — Games where user burned time early and collapsed in endgame. E.g.: "You consistently reach the endgame with <30 seconds and make 3× more errors."
FR-TIMING-6
Hesitation Patterns — Recurring hesitation on specific position types; confidence patterns; improvement in decision speed over time.

3.9.3 Time-Based Recommendations

FR-TIMING-7
Personalized Time Management Coaching — Recommend time allocation strategies; suggest thinking heuristics; flag opening lines where user consistently overspends.

3.10 AI Coach / Conversational Interface

The AI Coach is a primary interface of ChessIQ — not an add-on, but a core way users interact with their chess intelligence.

3.10.1 Conversational Capabilities

FR-AICOACH-1
Pattern-Based Questions — "Why do I keep blundering in the middlegame?", "What openings fit my style?", "How has my play changed over the last 3 months?"
FR-AICOACH-2
Specific Game Questions — "Why was move 18 a mistake?", "Was I winning at any point?", "Why did I spend 5 minutes and still blunder here?"
FR-AICOACH-3
Comparative Analysis — "How does my blitz compare to rapid?", "Do I play better as White or Black?", "Am I improving in tactics?"
FR-AICOACH-4
Improvement Planning — "Create a training plan for my endgame weaknesses", "Recommend drills for my knight fork problem."
FR-AICOACH-5
Opening & Repertoire Guidance — "What opening should I play against 1.e4?", "Is the Sicilian right for me?"

3.10.2 AI Coach Intelligence Requirements

FR-AICOACH-6
Context-Aware Responses — Must reference user's actual patterns and data; include specific examples from user's games; avoid generic advice.
FR-AICOACH-7
Multi-Source Intelligence — Synthesizes: Pattern Engine outputs, Stockfish findings, historical game trends, time management patterns, phase-based performance.
FR-AICOACH-8
Conversational Memory — Maintain context within a session; reference prior questions; build upon established player profile.
FR-AICOACH-9
Actionable Recommendations — Specific and actionable; include direct drill links; prioritize by impact and feasibility.

3.10.3 Explanation Quality

FR-AICOACH-10
Human-Readable Explanations — Conversational language (not notation-heavy); analogies and conceptual explanations; depth adapted to user skill level.
FR-AICOACH-11
Evidence-Based Coaching — Cite specific evidence ("In 12 similar positions…"); show statistics; reference patterns by confidence level.

Non-Functional Requirements

4.1 Performance

  • Single batch analysis of ~25 rapid games: typically complete within 2–5 minutes under normal load
  • Dashboard load time (summary + games): under 1.5 seconds P95 on typical broadband

4.2 Scalability

  • 10K+ registered users
  • 1M+ games stored
  • Dozens of concurrent analysis jobs handled by Celery workers

4.3 Reliability & Resilience

  • Chess.com API unavailable: show human-readable error, suggest retry, enforce backend timeouts
  • Individual game analysis failure: mark analysis_failed, continue others, surface minimal error in logs only

4.4 Security & Privacy

  • No user passwords stored (MVP uses username + email only)
  • All database connections over TLS (Supabase)
  • User PGNs and analysis data are private and not visible to other users
  • API endpoints protected: rate limiting, payload size limits

UX & Interaction Notes

5.1 Landing Page

  • Clear value prop explanation
  • Simple form: Chess.com username, email, filters (count, time control, rated)
  • Primary CTA: "Get Started"
  • Inline status messages: "Creating account…", "Fetching your games…"

5.2 Dashboard Layout (Recommended Order)

  1. Top Bar — Greeting, username, quick navigation
  2. AI Coach Widget — Quick-access chat, recent conversation snippets, pattern alerts: "New insight: You struggle with rook endgames"
  3. Key Metrics / Performance Cards — Overall accuracy, ACPL, Win rate, Time management score
  4. Pattern Recognition Panel — Player archetype summary, top 3–5 recurring patterns, evolution indicators
  5. Move Quality / Phase Charts
  6. Coaching Insights — Pattern-driven, severity-ranked with drill links
  7. Time Management Insights — Time usage visualization, overthinking/impulsive alerts
  8. Fetched Games (collapsible)

5.3 Empty States

  • No games fetched: CTA to go back to landing or "Fetch your first games" button
  • No analysis yet: Explanation and prominent "Analyze All Games" button

Roadmap Considerations (Beyond MVP)

7.1 Platform Integrations

  • Lichess integration
  • YouTube → PGN extraction
  • PGN upload from external sources

7.2 AI Coach Enhancements

  • Voice interface for AI Coach conversations
  • Real-time analysis during games (browser extension)
  • AI-generated video explanations with voice narration
  • Multi-language coaching support

7.3 Pattern Recognition Expansion

  • Cross-referential pattern analysis (compare with similar-rated players)
  • Opening repertoire optimization based on pattern data
  • Predictive performance modeling
  • Tournament preparation mode (analyze opponent patterns)

7.4 Training & Curriculum

  • Richer training curriculum and spaced repetition
  • AI-generated custom puzzle sets based on exact patterns
  • Opening repertoire builder with pattern-based recommendations
  • Integration with ChessReps for pattern-specific training lines

7.5 Social & Community

  • Social / sharing features (share reports, challenge friends)
  • Coach-student mode (human coach reviews AI insights)
  • Community pattern comparisons ("Players like you improve by…")

7.6 Mobile & Accessibility

  • Mobile-optimized or native mobile client
  • Offline mode with sync
  • Push notifications for new pattern insights
  • Accessibility improvements (screen reader optimization)
