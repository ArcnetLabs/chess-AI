Executive Summary

ChessIQ implements a hybrid memory and retrieval architecture that combines structured relational storage (Supabase Postgres) with semantic vector retrieval (pgvector) and ephemeral caching (Redis). This three-tier approach enables both transactional integrity for chess data and semantic intelligence for AI coaching, while maintaining grounded truth through Stockfish analysis.

Core Innovation: ChessIQ's AI context system is not generic document RAG. It is structured chess intelligence retrieval — a domain-specific architecture that grounds LLM explanations in verified chess analysis, behavioral patterns, and longitudinal player memory.

Key Principle: The LLM never invents chess truth. Stockfish provides objective analysis. The retrieval system assembles contextual intelligence. The LLM explains, personalizes, and coaches.

1.2 The Grounding Principle

  • Chess.com is the external source of raw game data: PGNs, ratings, timestamps, metadata
  • Stockfish is the source of objective chess truth: position evaluation, best moves, tactical detection
  • The LLM's constrained role: Explain Stockfish output, personalize to player level, summarize patterns, generate recommendations, manage conversational flow
Critical Boundary

The LLM never says "This position is +2.3 for White" without Stockfish providing that evaluation. The LLM explains: "The engine evaluates this as favorable for White because of the active rook and passed pawn."

2.2 Tier 2: pgvector / Vector Storage — Semantic Memory Layer

Critical Clarification

Vector DB is NOT transactional storage, NOT the source of truth, and NOT for structured queries. It IS a semantic search layer for finding conceptually similar information based on meaning.

3.4 Semantic Memory Retrieval

Scope: Conceptually similar situations across all user history · Storage: semantic_memory with pgvector

python
class SemanticMemory:
    async def retrieve_similar_memories(
        self, user_id: int, query: str, memory_types: List[str], top_k: int = 5
    ) -> List[SemanticMemory]:
        # 1. Generate query embedding
        query_embedding = await self.embedder.embed(query)

        # 2. Vector similarity search
        results = await self.db.query("""
            SELECT id, memory_type, content_summary, metadata,
                   1 - (embedding <=> %s::vector) as similarity_score
            FROM semantic_memory
            WHERE user_id = %s AND memory_type = ANY(%s)
            ORDER BY embedding <=> %s::vector
            LIMIT %s
        """, (query_embedding, user_id, memory_types, query_embedding, top_k))

        # 3. Filter by similarity threshold
        return [r for r in results if r.similarity_score > 0.75]

3.5 Memory Ranking & Pruning

python
class MemoryRanker:
    def rank_memories(self, query, memories, profile) -> List[RankedMemory]:
        ranked = []
        for memory in memories:
            score = 0.0
            # Factor 1: Recency (exponential decay, 30-day half-life)
            days_old = (datetime.now() - memory.created_at).days
            score += recency_weight * exp(-days_old / 30)

            # Factor 2: Semantic similarity
            score += similarity_weight * memory.similarity_score

            # Factor 3: Pattern severity
            severity_multiplier = {
                'critical': 1.5, 'significant': 1.2,
                'developing': 1.0, 'historical': 0.8
            }.get(getattr(memory, 'severity', None), 1.0)
            score += severity_weight * severity_multiplier

            # Factor 4: User engagement
            if hasattr(memory, 'access_count'):
                score += engagement_weight * min(memory.access_count / 10, 1.0)

            ranked.append(RankedMemory(memory=memory, relevance_score=score))

        return sorted(ranked, key=lambda x: x.relevance_score, reverse=True)

4.2 Structured Retrieval (SQL Layer)

python
class StructuredRetriever:
    def retrieve_relevant_patterns(self, user_id: int, intent: Intent) -> List[Pattern]:
        keywords = intent.extract_keywords()
        return self.db.query("""
            SELECT * FROM player_patterns
            WHERE user_id = %s
              AND (
                pattern_type = ANY(%s)
                OR pattern_subtype ILIKE ANY(%s)
                OR pattern_description ILIKE ANY(%s)
              )
              AND confidence_score > 0.6
            ORDER BY
              CASE severity
                WHEN 'critical' THEN 1
                WHEN 'significant' THEN 2
                WHEN 'developing' THEN 3
                ELSE 4
              END, confidence_score DESC
            LIMIT 5
        """, (user_id, intent.pattern_types, keywords, keywords))

5.2 Prompt Assembly

python
class PromptAssembler:
    def assemble(self, context: CoachContext, query: str, token_budget: int = 4000) -> Prompt:
        sections = []
        current_tokens = 0

        # 1. System prompt (~200 tokens, always included)
        sections.append(self.system_prompt())

        # 2. Player profile (~300 tokens, high priority)
        if context.profile:
            sections.append(self.format_profile(context.profile))

        # 3. Relevant patterns (~800 tokens, ranked by severity)
        if context.patterns:
            sections.append(self.format_patterns(context.patterns, max_tokens=800))

        # 4. Semantic memories (~500 tokens, if token budget allows)
        if context.semantic_memories and current_tokens < token_budget * 0.7:
            sections.append(self.format_semantic_memories(context.semantic_memories[:3]))

        # 5. Conversation history (~600 tokens, if space)
        if context.conversation_history and current_tokens < token_budget * 0.85:
            sections.append(self.format_history(context.conversation_history, max_tokens=600))

        # 6. Current question (always included)
        sections.append(f"\n## Current Question\nPlayer: {query}\n\nCoach: ")

        return Prompt(content="\n\n".join(sections))

6.3 Chunking Strategy

python
class ConversationChunker:
    def chunk_conversation(self, messages: List[Message], chunk_size: int = 512) -> List[Chunk]:
        chunks, current_chunk, current_tokens = [], [], 0
        for i, msg in enumerate(messages):
            msg_tokens = self.estimate_tokens(msg.content)
            if current_tokens + msg_tokens > chunk_size or self.is_topic_shift(messages, i):
                if current_chunk:
                    chunks.append(self.create_chunk(current_chunk))
                current_chunk, current_tokens = [msg], msg_tokens
            else:
                current_chunk.append(msg)
                current_tokens += msg_tokens
        if current_chunk:
            chunks.append(self.create_chunk(current_chunk))
        return chunks

6.4 Embedding Lifecycle Management

python
class EmbeddingLifecycleManager:
    async def refresh_embeddings(self, user_id: int):
        """Refresh embeddings for users with updated patterns."""
        updated_patterns = await self.db.query("""
            SELECT p.* FROM player_patterns p
            LEFT JOIN semantic_memory sm ON p.id = sm.source_id
            WHERE p.user_id = %s
              AND (sm.id IS NULL OR p.updated_at > sm.created_at)
        """, (user_id,))
        for pattern in updated_patterns:
            summary = self.summarizer.summarize_pattern(pattern)
            embedding = await self.embedder.embed(summary)
            await self.vector_db.upsert(user_id=user_id, memory_type='pattern',
                                        source_id=pattern.id,
                                        content_summary=summary, embedding=embedding)

    async def prune_stale_embeddings(self, user_id: int, max_age_days: int = 365):
        await self.db.execute("""
            DELETE FROM semantic_memory
            WHERE user_id = %s AND memory_type = 'pattern'
              AND created_at < NOW() - INTERVAL '%s days'
              AND source_id NOT IN (
                SELECT id FROM player_patterns
                WHERE user_id = %s AND last_seen_at > NOW() - INTERVAL '90 days'
              )
        """, (user_id, max_age_days, user_id))

9.3 Post-Generation Verification Layer

class HallucinationGuard:
    async def verify_response(self, response: str, context: CoachContext) -> VerificationResult:
        issues = []

        # 1. Verify evaluation claims
        eval_matches = re.findall(r'([+-]\d+\.\d+)', response)
        for claimed_eval in eval_matches:
            if not self.eval_in_context(claimed_eval, context):
                issues.append(HallucinationIssue(type="eval_mismatch",
                                                  claim=claimed_eval, severity="high"))

        # 2. Verify pattern statistics
        stat_matches = re.findall(r'(\d+) (times|games|occurrences)', response)
        for number, unit in stat_matches:
            if not self.stat_in_context(int(number), unit, context):
                issues.append(HallucinationIssue(type="stat_mismatch",
                                                  claim=f"{number} {unit}", severity="medium"))

        # 3. Verify move suggestions
        move_matches = re.findall(r'([KQRBN]?[a-h]?[1-8]?x?[a-h][1-8]=?[QRBN]?)', response)
        for move in move_matches:
            if not self.move_verified(move, context):
                issues.append(HallucinationIssue(type="unverified_move",
                                                  claim=move, severity="medium"))

        return VerificationResult(
            is_valid=len(issues) == 0, issues=issues,
            corrected_response=self.correct_response(response, issues) if issues else response
        )

class RetrievalOptimizer:
    async def fast_retrieve(self, user_id: int, query: str) -> CoachContext:
        cache_key = f"context:{user_id}:{hash(query)}"

        # 1. Check cache first
        cached = await self.cache.get(cache_key)
        if cached:
            return CoachContext.deserialize(cached)

        # 2. Parallel retrieval
        profile, patterns, vector_results = await asyncio.gather(
            self.get_profile_cached(user_id),
            self.get_patterns_cached(user_id),
            self.vector_search(query, user_id)
        )

        context = CoachContext(
            profile=profile,
            patterns=patterns[:5],           # Top 5 only
            semantic_memories=vector_results[:3]   # Top 3 only
        )

        # 3. Cache for 5 minutes
        await self.cache.set(cache_key, context.serialize(), ttl=300)
        return context

  3. Hybrid Retrieval — SQL for structured queries, Vector for semantic search, combined for optimal context
  4. Multi-Horizon Memory — Short-term (session), medium-term (30–90 days), long-term (archetype), semantic (cross-history)
