# Legacy Docs

Generated and transient documents that have no long-term architectural value. Kept for audit trail purposes only — do not reference these in active development or onboarding.

## Contents

| Category | Files |
|----------|-------|
| Phase completion reports | `backend_PHASE*_IMPLEMENTATION_COMPLETE.md`, `PHASE1_*.md` |
| Planning notes (superseded) | `backend_PHASE*_PLAN.md`, `backend_PHASE2_MOVE_RECOMMENDATIONS_PLAN.md` |
| Generated status reports | `CELERY_STATUS_REPORT.md`, `RATE_LIMIT_STATUS_REPORT.md`, `REDIS_CACHE_STATUS_REPORT.md`, `CELERY_DEMO_RESULTS.md`, `backend_TEST_COVERAGE_REPORT.md` |
| One-off fix/patch notes | `BUGFIX_FETCH_GAMES.md`, `QUICK_FIX_WINDOWS_STOCKFISH.md` |
| Generated summaries | `IMPLEMENTATION_SUMMARY.md`, `OPTIMIZATION_SUMMARY.md` |
| Design tool artifacts | `STITCHES_DESIGN_PROMPT.md` |

## What lives in `docs/archive/` instead

Files with genuine reference value (setup guides, implementation details, user flows, Stockfish/Celery manuals) remain one level up in `docs/archive/`.

Active architectural, deployment, product, and strategy documentation lives in the top-level `docs/` subdirectories — never here.
